Product overview
GreenCode — AST-Powered Sustainable Static Analyzer
A Java-based static analysis tool that parses source code into an Abstract Syntax Tree, detects "energy smell" patterns, and scores code on algorithmic efficiency and estimated carbon impact.
MVP scope Java backend React frontend ESG-oriented
Goals
Business goal
Enable developers and enterprises to measure and reduce the "carbon debt" of their codebases before deployment, supporting ESG commitments and reducing cloud compute costs.
Product goal
Ship an MVP that detects the top 3 energy smell categories via AST analysis and surfaces a Green Score with actionable refactor suggestions — no ML, no cloud inference required.
Target users
🧑‍💻 Developer / student
Wants to learn the energy cost of their logic and write better code.
🏢 Tech lead / CIO
Needs to measure and reduce carbon debt across large codebases for ESG reporting.
🔬 GreenTech researcher
Quantifies how software design patterns affect power consumption and hardware degradation.
MVP feature list — prioritised
P0
AST parsing engine
Parse uploaded Java source files into an AST using JavaParser. Foundation for all analysis.
P0
Energy smell detection (3 types)
Detect: (1) O(n²) nested loops, (2) loop-invariant redundant computation, (3) busy-wait / empty polling loops.
P0
Green Score (A+ to F)
Single weighted grade derived from smell count, severity, and code size. Shown in dashboard on file upload.
P0
Refactor suggestions
For each detected smell, show a minimal before/after code snippet with Big O annotation.
P1
CO₂ estimation
Theoretical model: "If this function runs 1M times on a standard AWS t3.medium, it emits ~X g CO₂." Based on published cloud emission coefficients.
P1
Data flow graph (D3.js)
Visual execution-path graph of flagged code blocks. Nodes = statements, edges = data flow.
P2
Multi-file / project scan
Upload a zip; aggregate scores per class and at project level. Out of MVP scope.

Energy smells — detection rules
Nested loop
O(n²) or worse
Flag: two or more nested ForStatement / WhileStatement nodes in the AST. Suggest: HashMap / sort-based O(n log n) approach.
Loop invariant
Redundant computation
Flag: method call or expression inside a loop whose operands are not modified in the loop body. Suggest: hoist outside loop.
Busy wait
Empty / polling loop
Flag: loop body is empty or contains only a condition check with no yield/sleep. Suggest: Thread.sleep(), wait/notify, or event-driven pattern.
Architecture — MVP
Browser UI
React + D3.js — file upload, Green Score dashboard, smell list, data flow graph
REST API
Spring Boot — exposes POST /analyze endpoint, returns JSON analysis result
Analysis engine
Java — JavaParser AST traversal, smell detectors (Visitor pattern), Green Score calculator, CO₂ estimator
Suggestions DB
Static JSON / YAML — maps smell type → refactor snippet + Big O annotation (no ML required for MVP)
Core logic constraint: AST parsing, smell detection, scoring, and CO₂ calculation are Java-only (no Python, no Node.js in the backend).
