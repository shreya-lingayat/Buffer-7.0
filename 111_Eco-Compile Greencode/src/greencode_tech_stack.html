GreenCode — recommended tech stack

MVP / hackathon build · Java backend · React frontend

Frontend
React 19recommended
Component model maps cleanly to score dashboard + smell list. Huge ecosystem, easy Vite setup for hackathon speed.
Tailwind CSS v4
Utility-first — no custom CSS needed for MVP. Works out of the box with Vite. No design system required.
React Flow
Easier than D3 for node/edge graphs. Renders the data-flow graph with zero layout math. Drop-in component.
Backend
Spring Boot 3.xrecommended
Minimal boilerplate REST API in Java. Single @PostMapping /analyze endpoint. Satisfies Java constraint natively.
Maven
Standard Java build tool. JavaParser and Spring Boot both have first-class Maven support — no configuration friction.
AST parsing & analysis engine
JavaParser 3.xrecommended
Purpose-built Java AST library. Visitor pattern built-in — maps 1:1 to the PRD's ForStatement / WhileStatement detectors. No ANTLR grammar needed.
Data persistence
Static JSON / YAMLrecommended
Suggestions DB is read-only for MVP. A bundled YAML file in the Spring Boot resources folder is zero-overhead — no DB setup required.
Deployment
Vercelfrontend
One vercel deploy from the React repo root. Free tier, instant CDN, zero config for Vite builds.
Railwaybackend
Deploys a Spring Boot fat JAR from a GitHub repo in <5 min. Free tier sufficient for hackathon/MVP traffic. Simpler than AWS for a Java API.
Request flow
Browser (Vercel)
User uploads .java file → React sends multipart/form-data to backend
Spring Boot (Railway)
POST /analyze receives file → passes to analysis engine
JavaParser engine
AST traversal → smell detectors → score calculator → suggestion lookup
JSON response
Returns { score, grade, smells[], co2estimate } → React renders dashboard
